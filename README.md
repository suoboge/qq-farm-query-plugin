# QQ农场查询插件

基于 NapCat 的 QQ 农场查询插件，支持扫码登录、农场状态查询、背包管理、成熟时间查看等功能。
交流群：1090049575
## 插件信息

- **名称**: QQ农场查询插件
- **作者**: 鲁班
- **版本**: 1.0.0
- **支持 NapCat 版本**: ≥ 4.14.0

## 功能列表

| 功能 | 命令 | 说明 |
|------|------|------|
| 扫码登录 | `登录` | 扫码登录 QQ 农场 |
| 农场状态 | `我的农场` | 查看农场土地状态 |
| 背包物品 | `我的背包` | 查看背包中的物品 |

| 种子列表 | `种子列表` | 查看所有种子ID（无需登录） |
| 收获作物 | `收获` | 收获所有成熟的作物 |
| 铲除作物 | `铲除` | 铲除枯萎的作物 |
| 种植作物 | `种植 种子ID` | 在空地上种植作物 |
| 升级土地 | `升级土地 土地ID` | 升级指定土地 |
 `扩展土地 土地ID` |扩展新土地|

## 部署说明

### 环境要求

- Node.js ≥ 18.0.0
- pnpm ≥ 8.0.0
- NapCat ≥ 4.14.0

### 安装步骤

1. **克隆项目**
```bash
git clone <项目地址>
cd napcat-plugin-template-main
```

2. **安装依赖**
```bash
pnpm install
```

3. **修改插件信息（可选）**

编辑 `package.json`，修改以下字段：
```json
{
    "name": "qq-farm-query-plugin",
    "description": "QQ农场查询插件 - 扫码登录后查看农场状态、背包、成熟时间等",
    "author": "鲁班"
}
```

4. **构建插件**
```bash
pnpm run build
```

5. **部署**

#### 方式一：一键部署（推荐）
确保 NapCat 端已安装并启用 `napcat-plugin-debug` 插件：
```bash
pnpm run deploy
```

#### 方式二：手动部署
将 `dist/` 目录的内容复制到 NapCat 的插件目录：

- Windows: `NapCat/NapCat.exe/plugins/`
- Linux/Mac: `napcat/plugins/`

6. **重载插件**

在 NapCat 控制台发送：
```
重载插件
```

或重启 NapCat 服务。

## 注意事项

1. **首次使用需要扫码登录**，登录状态会保存在内存中，重启 NapCat 后需要重新登录

2. **每个QQ需要单独扫码登录**，插件按 QQ 号区分不同用户的登录状态

3. **命令前缀支持模糊匹配**，如发送"我的农场"或"农场"都可以触发

4. **部分操作可能有延迟**，如种植、收获等操作需要等待服务器响应

5. **冷却时间**，同一命令在同一个群内有冷却时间限制，默认30秒

6. **仅支持群聊和私聊**，插件会自动识别消息类型并作出响应
